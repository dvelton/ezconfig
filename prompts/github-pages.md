# Deploy to GitHub Pages

Paste this into your AI tool when you're ready to deploy a static site to GitHub Pages.

---

I have a project in this directory that I want to deploy as a website using GitHub Pages. Here's what I need you to do:

1. Look at what I've built and confirm it's a static site (HTML/CSS/JS, no server needed)
2. Make sure there's an index.html at the root (or tell me if things need to be reorganized)
3. Create a GitHub repository for it if one doesn't exist
4. Push the code to GitHub
5. Set up GitHub Pages deployment — either using a GitHub Actions workflow or by configuring the repo to serve from the main branch
6. Tell me the URL where my site will be live

If my project uses a build step (like React with Vite, or any framework that compiles), set up a GitHub Actions workflow that builds the project and deploys the output to Pages.

Keep everything free. Walk me through anything that needs a manual step (like enabling Pages in the repo settings).
