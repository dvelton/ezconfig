Here's what I need you to do:

1. Look at my project and figure out what data it needs to store and what features it needs
2. Walk me through creating a free Supabase project at supabase.com
3. Design the database tables based on what my app needs — create them in Supabase
4. If I need user authentication (login/signup), set up Supabase Auth:
   - Email/password signup
   - Social login (Google, GitHub) if I want it
   - Row-level security so users can only see their own data
5. Connect my frontend to Supabase using the JavaScript client library
6. If I need file uploads (images, documents), set up Supabase Storage
7. Deploy the frontend somewhere free (Vercel, Netlify, or GitHub Pages) and make sure it connects to Supabase in production
8. Set up environment variables properly (don't put API keys directly in the code)

Everything should be on the free tier. Explain any limits (how much storage, how many users, etc.) so I know when I'd need to upgrade.
